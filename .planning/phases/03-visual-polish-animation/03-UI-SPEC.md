## Animation Contracts

### GSAP Loading

| Property | Value |
|----------|-------|
| Package | `gsap` (npm, latest 3.x) |
| Plugin | `ScrollTrigger` (bundled with gsap) |
| Loading | Astro `client:idle` or inline `<script>` with `type="module"` |
| Registration | `gsap.registerPlugin(ScrollTrigger)` on script init |
| Reduced motion | `if (window.matchMedia('(prefers-reduced-motion: reduce)').matches)` -- skip all animations, remove `.js-animate` class so content is visible |

### Progressive Enhancement Contract

| State | Behavior |
|-------|----------|
| JS disabled | All content visible. No animations. Diagrams show final static state (all nodes/lines at full opacity). Dividers visible. Constellation visible. |
| JS enabled, reduced motion | All content visible immediately. No animations play. `.js-animate` class removed. |
| JS enabled, motion OK | Animations play as specified below. |

**Implementation approach:** Content starts visible in HTML. A small inline `<script>` in `<head>` (before GSAP loads) adds `.js-animate` class to `<html>`, which activates CSS rules that hide animated elements (`opacity: 0; transform: translateY(30px)`). If JS never runs (disabled), the class is never added, and content stays visible. GSAP then animates elements to their final visible state.

### Hero Load Animation (D-05, D-08, D-09, D-10, D-11)

| Step | Target | Property | From | To | Duration | Delay | Ease |
|------|--------|----------|------|----|----------|-------|------|
| 1 | `[data-hero-headline]` | opacity, y | 0, 20px | 1, 0 | 0.5s | 0s | `power2.out` |
| 2 | `[data-hero-paragraph]` | opacity, y | 0, 20px | 1, 0 | 0.5s | 0.3s | `power2.out` |
| 3 | Fano lines `[data-fano-line]` | strokeDashoffset | (pathLength) | 0 | 1.2s | 0.8s | `power1.inOut` |
| 4 | Fano nodes `[data-fano-node]` | opacity, scale | 0, 0.5 | 1, 1 | 0.4s | stagger 0.08s after lines complete | `back.out(1.4)` |
| 5 | Fano subalgebra glow | filter brightness | 1 | 1.5 then 1 | 0.3s per subalgebra | sequential, 0.15s gap | `power1.inOut` |

Total hero sequence: approximately 2.5s. Plays once on page load. No replay.

**Fano draw-on detail:**
- Each `<line>` and the inscribed `<circle>` need `stroke-dasharray` set to their total path length, and `stroke-dashoffset` set to the same value (fully hidden).
- GSAP animates `strokeDashoffset` to 0 (fully drawn).
- Lines animate in order: Line 1 through Line 6, then inscribed circle (Line 7). Stagger 0.1s between lines. Total line draw: ~1.2s.
- After lines complete, nodes fade in with `back.out(1.4)` for a slight overshoot (scale 0.5 > 1).
- After nodes appear, each subalgebra line briefly brightens (filter or opacity pulse from 0.85 to 1 to 0.85) in sequence. 7 pulses at 0.15s gap = ~1.05s.

## Performance Budget

| Metric | Target | Rationale |
|--------|--------|-----------|
| GSAP bundle | < 30KB gzipped | GSAP core + ScrollTrigger. Tree-shake unused plugins. |
| Total JS payload | < 35KB gzipped | GSAP is the only client-side JS. |
| Constellation SVG | < 5KB | Simple geometry, no filters, no glow. |
| Time to Interactive | < 2s on 4G | GSAP loads via `client:idle`, does not block TTI. |
| Largest Contentful Paint | No regression from Phase 2 | All content renders server-side before GSAP loads. |
| Cumulative Layout Shift | 0 | Animated elements use `opacity` and `transform` only (compositor-friendly). No layout-shifting properties. |

**Animation performance rules:**
- Only animate `opacity` and `transform` (GPU-composited properties).
- SVG `stroke-dashoffset` is acceptable (does not trigger layout).
- Never animate `width`, `height`, `margin`, `padding`, or `top`/`left`.
- `will-change: transform, opacity` on animated elements during animation, removed after completion.
- Constellation SVG has no filter effects (unlike Fano/trie glow which are already present from Phase 2).

Source: REQUIREMENTS.md DEPL-03 (Lighthouse >= 90), CONTEXT.md D-18
