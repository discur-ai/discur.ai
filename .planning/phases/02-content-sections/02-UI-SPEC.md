## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Tight element spacing, equation label gaps |
| md | 16px | Default paragraph spacing (`mb-4`), table cell padding (`px-4 py-3`) |
| lg | 24px | Heading margin-bottom (`mb-6`), section internal grouping |
| xl | 32px | Hero top padding above headline (`pt-32` = 128px, handled by hero section) |
| 2xl | 48px | Major content group separation within sections |
| section | `clamp(4rem, 10vw, 10rem)` | Between-section vertical spacing (existing `--spacing-section` token) |

Exceptions:
- Hero section uses `pt-32` (128px) to clear the fixed header (56px) plus visual breathing room. This is a layout constraint, not a spacing token violation.
- Section component applies responsive horizontal padding: `px-4` (16px) / `sm:px-6` (24px) / `lg:px-8` (32px).

Source: global.css `--spacing-section`, Section.astro padding classes, existing index.astro patterns

### Hero Layout

```
Desktop (>= 640px):               Mobile (< 640px):
+---------------------------+      +------------------+
| [Headline text]  [Fano   |      | [Headline text]  |
| [Brief para]      plane] |      | [Brief paragraph]|
+---------------------------+      | [Fano plane]     |
                                   +------------------+
```

- Desktop: CSS Grid or Flexbox, text left (60% width), Fano plane right (40% width)
- Mobile: Stack vertically, text first, Fano plane below (centered, max-width 280px)
- Hero headline: `text-glyph` (#E8D5A3), Cinzel, responsive sizes `text-4xl sm:text-5xl lg:text-6xl`
- No equation in hero (D-03)

- Use `Section` with `fullWidth` prop for the trie diagram container
- Trie SVG: `width="100%" max-width="800px"`, centered with `mx-auto`

- Surrounding text paragraphs use standard `max-w-4xl` Section width

| Property | Value |
|----------|-------|
| viewBox | `0 0 400 360` |
| preserveAspectRatio | `xMidYMid meet` |
| Container max-width | 400px (desktop), 280px (mobile) |
| Node radius | 8px |

| Node filter | Gaussian blur glow (stdDeviation: 4, gold colormatrix) |

| Inscribed circle | Passes through 3 edge midpoints, stroke matches Line 7 color |

### Trie Structure (`TrieStructure.astro`)

| Property | Value |
|----------|-------|
| viewBox | `0 0 800 400` (landscape) |
| preserveAspectRatio | `xMidYMid meet` |
| Container max-width | 800px, centered |
| Node radius | 12px (root), 10px (children), 8px (grandchildren) |
| Node fill | `#C4A35A` (sigil) |
| Node filter | Same glow filter as Fano plane |
| Edge stroke | `#8A8680` (chalk-muted), stroke-width 1.5px |
| Edge labels | `font-display text-xs text-sigil`, positioned at edge midpoints (S_1, S_2, etc.) |
| Structure | Root + 3 children + 3 grandchildren (thesis Figure 1) |
| role | `img` |
| aria-label | "Octonionic trie structure showing three levels of hierarchical memory nodes" |

### SVG Performance Constraints

- Glow filter applied ONLY to circle nodes (max 17 total: 7 Fano + 7 trie + root)
- No glow on line/path elements
- stdDeviation capped at 4 (do not exceed 5)
- Explicit filter region bounds: `x="-50%" y="-50%" width="200%" height="200%"`

## Copywriting Contract

### Voice Rules

| Rule | Specification |
|------|---------------|
| Person | Third-person active voice. Never "we." (D-16) |
| Tone | Anthropic research blog style. Declarative, specific, neutral. (D-23) |
| Em dashes | Zero on entire page. Use commas, periods, or parentheses. (D-17 H5) |
| Superlatives | Zero unearned superlatives: no "revolutionary", "groundbreaking", etc. (D-17 H3) |
| Hedging | Zero hedge phrases: no "it's worth noting", "interestingly", etc. (D-17 H4) |
| Binary contrasts | No "this isn't X, it's Y" constructions. State the positive claim. (D-17 H1) |
| Staccato fragments | No standalone fragments for dramatic effect. (D-17 H2) |
| Depth | As much as needed per section. No arbitrary word count limit. (D-19) |
| Audience | Technical generalists. Layered so ML researchers are rewarded by equations. (D-20) |
| Section headings | Minimal, single words: "Mechanism", "Architecture", "Invariants", "Results", "Comparison" (D-24) |

### Copy Elements

| Element | Copy |
|---------|------|
| Hero headline | "A self-organizing memory that replaces gradient descent with octonionic algebra" (D-03, or equivalent architecture-first framing) |
| Hero paragraph | 2-3 sentences. What the trie does and how. No equation. |
| Section heading style | Single word, Cinzel, sigil color, uppercase tracking |
| Primary CTA | "Read the paper" (links to GitHub under discur-ai) |
| Contact email | research@discur.ai (D-12) |
| GitHub link | github.com/discur-ai (D-12, exact repo URL TBD, use placeholder) |
| Empty state | Not applicable (static content page, no dynamic data) |
| Error state | Not applicable (static site, no server errors) |
| Destructive actions | None |

### Anti-Slop Checklist (CONT-10)

Every paragraph of copy MUST pass ALL tiers before committing. Full checklist in RESEARCH.md. Summary of enforcement:

- **Tier 1 (Hard Rules):** H1-H5, zero tolerance, string-searchable
- **Tier 2 (Structural):** S1-S8, AI detection signals
- **Tier 3 (Word-Level):** W1-W11, banned words/phrases (82 specific items)
- **Tier 4 (Tone):** T1-T6, qualitative pass/fail tests
- **Tier 5 (Formatting):** F1-F4, formatting tells

Post-phase copy audit planned per D-25.

Source: CONTEXT.md D-16 through D-26, RESEARCH.md Anti-Slop Comprehensive Checklist

## Interaction Contracts

This phase has zero client-side JavaScript. All content is static HTML rendered at build time.

| Interaction | Implementation |
|-------------|----------------|
| Scroll | Native browser scroll. No scroll hijacking. No scroll-triggered animations (Phase 3). |
| Links | Standard anchor tags. `hover:text-sigil transition-colors` for hover state. |
| Table overflow | `overflow-x-auto` on table container for mobile. Native touch scrolling. |
| Equation overflow | `overflow-x-auto overflow-y-hidden` on equation container (existing Equation.astro). |
| SVG scaling | `viewBox` + `preserveAspectRatio`. No JavaScript resize handlers. |
| Text selection | `::selection { background-color: sigil; color: void; }` (existing global.css). |

No modals, no tooltips, no accordions, no client-side state. All animation deferred to Phase 3.

## Responsive Breakpoints

Using Tailwind's default breakpoints (established in Phase 1):

| Breakpoint | Width | Key Layout Changes |
|------------|-------|-------------------|
| Default (mobile) | < 640px | Single column. Hero stacks vertically. Results cards stack. Tables scroll horizontally. Fano plane at 280px max. |
| `sm` | >= 640px | Hero side-by-side (text/Fano). Results cards 2-column grid. Body text scales to `text-xl`. |
| `lg` | >= 1024px | Hero heading scales to `text-6xl`. Section padding increases to `px-8`. |

Content max-width: `max-w-4xl` (896px) for all text sections. Trie diagram uses `fullWidth` with `max-w-[800px] mx-auto`.

## Accessibility

| Concern | Implementation |
|---------|----------------|
| Math equations | `role="math"` + `aria-label` on every Equation component (existing pattern) |
| SVG diagrams | `role="img"` + `aria-label` on each SVG element |
| Color contrast | chalk (#E0DCD4) on void (#0A0A0F) = 13.2:1 ratio (exceeds WCAG AAA). sigil (#C4A35A) on void (#0A0A0F) = 7.1:1 ratio (exceeds WCAG AA). |
| Heading hierarchy | h1 (hero) then h2 (each section), no skipped levels |
| Link purpose | All links have descriptive text ("Read the paper", "research@discur.ai") |
| Table structure | Semantic `<table>`, `<thead>`, `<th>`, `<tbody>`, `<td>` elements |
| Reduced motion | Not applicable in Phase 2 (no animations). Phase 3 will add `prefers-reduced-motion`. |
