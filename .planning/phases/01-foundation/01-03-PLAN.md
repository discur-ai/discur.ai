    1. Create `src/components/math/Equation.astro` (display-mode, block equations):
       - Import `katex` from `'katex'` in the frontmatter block (this is server/build-time only)
       - Props interface: `{ latex: string; label?: string; ariaLabel?: string }`
       - Call `katex.renderToString(latex, { displayMode: true, throwOnError: false, output: 'htmlAndMathml' })` in frontmatter
       - Template wraps output in `<figure>` with `role="math"` and `aria-label` for accessibility
       - CRITICAL per FOUN-03: The inner scroll container MUST have BOTH `overflow-x: auto` AND `overflow-y: hidden`. KaTeX's vertical alignment hack produces spurious vertical overflow when only overflow-x is set. Also set `max-w-full`.
       - Optional `label` prop renders as `<figcaption>` with equation number (e.g., "(1)")
       - Use `set:html` directive to inject the KaTeX output (no client-side processing)
       - Full template:
         ```astro
         ---
         import katex from 'katex';

         interface Props {
           latex: string;
           label?: string;
           ariaLabel?: string;
         }

         const { latex, label, ariaLabel } = Astro.props;
         const html = katex.renderToString(latex, {
           displayMode: true,
           throwOnError: false,
           output: 'htmlAndMathml',
         });
         ---

         <figure
           class="equation-container my-8 flex items-center justify-between"
           role="math"
           aria-label={ariaLabel}
         >
           <div class="equation-scroll overflow-x-auto overflow-y-hidden py-2 max-w-full">
             <Fragment set:html={html} />
           </div>
           {label && (
             <figcaption class="equation-label ml-4 shrink-0 font-mono text-sm text-sigil">
               ({label})
             </figcaption>
           )}
         </figure>
         ```

    2. Create `src/components/math/InlineMath.astro` (inline equations within text):
       - Import `katex` from `'katex'` in frontmatter
       - Props interface: `{ latex: string; ariaLabel?: string }`
       - Call `katex.renderToString(latex, { displayMode: false, throwOnError: false, output: 'htmlAndMathml' })` in frontmatter
       - Template is a `<span>` with `role="math"`, `aria-label`, and `set:html`
       - Full template:
         ```astro
         ---
         import katex from 'katex';

         interface Props {
           latex: string;
           ariaLabel?: string;
         }

         const { latex, ariaLabel } = Astro.props;
         const html = katex.renderToString(latex, {
           displayMode: false,
           throwOnError: false,
           output: 'htmlAndMathml',
         });
         ---

         <span
           class="inline-math"
           role="math"
           aria-label={ariaLabel}
           set:html={html}
         />
         ```

    CRITICAL ANTI-PATTERNS:
    - NEVER import katex in a `<script>` tag -- only in the `---` frontmatter block
    - NEVER use `client:load` or `client:visible` on these components
    - ALWAYS use both `overflow-x: auto` AND `overflow-y: hidden` together on the equation scroll container
